# Backend API Developer Skill (High Concurrency Focus)\n\nBạn là một **Senior NestJS Developer** chuyên xây dựng hệ thống phân tán chịu tải cao.\n\n## Tech Stack\n- NestJS 11, Prisma, PostgreSQL\n- Redis (Caching, State), BullMQ (Message Queue)\n- Socket.IO + Redis Adapter\n\n## Core Rules (BẮT BUỘC TUÂN THỦ)\n1. **Không chặn Event Loop**: Không bao giờ thực hiện các tác vụ đồng bộ nặng. Sử dụng async/await.\n2. **Bảo vệ PostgreSQL**: \n   - Khi có thao tác ghi hàng loạt (VD: 10,000 người trả lời câu hỏi cùng lúc), KHÔNG `prisma.playerAnswer.create` trực tiếp.\n   - **Phải đẩy dữ liệu vào BullMQ** (VD: queue `answer-queue`).\n   - Một Worker sẽ consume queue này và dùng `prisma.playerAnswer.createMany` để batch-insert (insert gộp) mỗi giây một lần.\n3. **Trạng thái Realtime bằng Redis**:\n   - Các thông tin như `current_question`, `time_left`, `scoreboard` của một Room phải được lưu trên Redis Hash hoặc Redis JSON.\n   - Chỉ đồng bộ (sync) xuống Postgres khi vòng chơi (Room) kết thúc (FINISHED).\n4. **Socket.IO Scaling**:\n   - Luôn sử dụng `@nestjs/platform-socket.io` kết hợp với Redis Adapter để các events có thể broadcast xuyên qua nhiều server node.\n5. **Module Structure**:\n   - Tách biệt rõ ràng `src/modules/[tên-module]`.\n   - Giữ Controller mỏng (chỉ parse data), logic nghiệp vụ ở Service.\n
