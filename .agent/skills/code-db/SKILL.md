# Database Modeler Skill (PostgreSQL & Prisma)\n\nBạn là chuyên gia thiết kế cơ sở dữ liệu và tối ưu hóa query cho bài toán 10k CCU.\n\n## Quy tắc thiết kế Schema\n1. **Naming Convention**: Table phải là snake_case số nhiều (ví dụ `@@map(\"player_answers\")`). Các trường trong table là snake_case (ví dụ `time_answered`, `@map(\"time_answered\")`).\n2. **Indexing**: \n   - Phải tạo `@@index` cho các khóa ngoại (Foreign Keys) thường xuyên dùng để JOIN (như `quiz_id`, `room_id`, `player_id`).\n   - Các cột thường dùng để sort (như `score`) cũng cần index: `@@index([roomId, score(sort: Desc)])`.\n3. **Connection Pooling**: Khi code service, phải lưu ý Prisma sẽ tạo connection pool. Hệ thống chịu tải cao cần cấu hình pool size hợp lý hoặc dùng PgBouncer ở tầng DB.\n4. **Tránh N+1 Queries**: Luôn dùng `include` hoặc `select` cẩn thận, tuyệt đối không dùng vòng lặp để gọi DB.\n5. **Batching**: Khi lưu lượng ghi cao, khuyến khích viết Prisma Raw Query `INSERT ... ON CONFLICT` nếu Prisma API không hỗ trợ UPSERT hàng loạt hiệu quả.\n
