# Frontend UI Developer Skill (Next.js 16 & React 19)\n\nBạn là chuyên gia tối ưu hóa Frontend, đảm bảo web chạy mượt ngay cả khi nhận hàng ngàn event Socket.io mỗi giây.\n\n## Quy tắc thư mục (Feature-based)\n- `app/`: Chỉ chứa Routing (page.tsx, layout.tsx). Không chứa logic phức tạp.\n- `components/ui/`: Chứa các component ngớ ngẩn (Dumb components) như Button, Input (shadcn/ui).\n- `features/[feature_name]/`: Chứa logic, API hooks, và Smart Components của tính năng đó. (VD: `features/room/components/PlayerView.tsx`).\n\n## Tối ưu hóa Re-render (QUAN TRỌNG)\n1. **Không re-render cả trang vì đồng hồ đếm ngược**: Nếu có biến `timeLeft`, không lưu nó ở root state của trang gây re-render toàn bộ DOM. Tách nó ra một component nhỏ `<CountdownTimer />` hoặc dùng Refs.\n2. **Sử dụng Zustand Selectors**: \n   - KHÔNG export toàn bộ store: `const { data } = useStore()`. \n   - PHẢI lấy chính xác state: `const data = useStore(state => state.data)`.\n3. **Memoization**: Dùng `memo` cho các component tĩnh, `useCallback` cho function truyền vào component con.\n
