# Tổng quan kiến trúc 10K CCU\n\n## 1. Bài toán\nKahoot-style: Khi Host bấm \"Next Question\", 10,000 điện thoại của Player nhận event gần như cùng lúc.\nTrong vòng 10 giây tiếp theo, 10,000 Player sẽ bấm chọn đáp án. Điều này tạo ra Spike Load (tải tăng vọt) 10,000 req/sec tới Server.\n\n## 2. Giải pháp kiến trúc\n- **Load Balancer**: Nginx/HAProxy chia tải vào 4-8 Node.js instances (NestJS).\n- **WebSocket Layer**: Các Node.js instances dùng Socket.io kết nối với nhau qua **Redis Adapter**. Host gửi event vào Node 1, Redis Adapter sẽ broadcast event ra toàn bộ các Node khác để push tới Player.\n- **Ingestion Layer**: Khi 10k người trả lời, request đi qua HTTP POST (hoặc Socket ack). Controller KHÔNG GHI VÀO DB POSTGRES. Controller đẩy payload `{ playerId, answerId, time }` vào **BullMQ (Redis)**.\n- **Worker Layer**: Một Worker chạy ngầm (cron hoặc queue processor) sẽ consume queue liên tục. Nó gom (batch) cứ 1000 answers thành 1 cụm, và gọi `prisma.playerAnswer.createMany` 1 lần.\n- **State Layer**: Toàn bộ điểm số, xếp hạng tạm thời (Leaderboard) được tính và lưu trong **Redis Sorted Sets** để query O(log(N)) cực nhanh, lấy ra Top 10 trong 1 mili-giây.\n
