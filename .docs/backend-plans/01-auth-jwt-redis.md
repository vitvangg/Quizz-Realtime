# Luồng Authentication & Rate Limiting\n\n## JWT & Sessions\n- Dùng Access Token (15 phút, stateless, không check DB).\n- Dùng Refresh Token (lưu Postgres `Session` hoặc Redis) để thu hồi khi cần.\n\n## Rate Limiting (Chống Spam)\n- Sử dụng `@nestjs/throttler` kết hợp Redis store.\n- Chặn hành vi 1 IP spam tạo 1000 players vào room ảo.\n- Giới hạn: API Join Room max 5 req/giây/IP.\n
