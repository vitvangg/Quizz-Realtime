# Realtime Engine (Socket.io + Redis)\n\n## Cấu trúc dữ liệu Redis cho 1 Room\n- Key: `room:{roomId}:state` (Hash) -> `{ status: 'PLAYING', currentQuestionIndex: 2, questionStartedAt: 168000000 }`\n- Key: `room:{roomId}:leaderboard` (Sorted Set) -> Lưu score của từng `playerId`.\n\n## Luồng Host Next Question\n1. Host gọi API `/rooms/:id/next-question`.\n2. Server lấy dữ liệu câu hỏi từ Postgres/Redis Cache.\n3. Server update Redis Hash: `questionStartedAt = Date.now()`, `currentQuestionIndex += 1`.\n4. Server emit qua Socket: `question_started` kèm payload câu hỏi (không có đáp án đúng).\n\n## Cách tính điểm (Score Engine)\n- Điểm phụ thuộc vào tốc độ trả lời.\n- `MaxScore = 1000`. `TimeLimit = 20s`.\n- Nếu trả lời sau 5s: `Score = 1000 * ((20 - 5) / 20) = 750`.\n- Việc chấm điểm sẽ do Worker làm khi nó bốc Job từ Queue ra. Worker sẽ check với đáp án đúng và cập nhật vào Redis Sorted Set `room:{roomId}:leaderboard` bằng `ZINCRBY`.\n
